# User Rules

1. **Design Alignment**: Always design according to the existing application's character and style.
2. **Scope Separation**: When working on design, do NOT modify functional code.
3. **Core Mechanics Protection**: Do NOT modify the game's core rules or mechanics.
4. **Modification Protocol**: If functional changes are necessary, you **MUST** first ask for permission in **Korean** explaining why the change is needed.
5. **Device Specific Design**: Treat iPhone and iPad designs as completely separate projects. Do NOT modify iPad design/code when working on iPhone, and vice-versa. When designing for the phone, do not touch the pad design, and when designing for the pad, only modify the pad design. Do not assume they share common design principles.
6. do not use snackbar
7. **Strict UI Consistency**: When implementing any UI/UX or design changes, you MUST strictly adhere to the shapes, colors, layout patterns, and overall aesthetic characteristics of the existing design. Do not introduce new or unaligned design paradigms that stray from the current look and feel of the app.
8. **Supabase Scope Restriction**: When modifying Supabase-related code or configurations, ONLY modify code related to the `link_your_area` game. Do NOT modify or affect data, rules, or logic for any other games.
